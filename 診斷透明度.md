# 透明度問題診斷

## ✅ 已確認修復

1. **預設透明度**: 1.0 (100%) ✓
2. **視窗創建時設置**: opacity: 1.0 ✓  
3. **載入設定後應用**: windowManager.setOpacity() ✓
4. **錯誤處理**: 如果載入失敗，fallback 到 1.0 ✓

## 🔍 如何確認視窗是否可見

### 方法 1：使用 Cmd+Tab
1. 按 `Cmd+Tab` 切換應用程式
2. 找到 VoiceInput
3. 放開按鍵

### 方法 2：使用 Mission Control
1. 按 `F3` 或三指上滑
2. 查看所有視窗
3. 找到 VoiceInput 視窗

### 方法 3：檢查 Dock
1. 查看 Dock 中是否有 VoiceInput 圖示
2. 右鍵點擊圖示
3. 選擇「顯示所有視窗」

### 方法 4：使用終端機檢查
\`\`\`bash
# 檢查進程
ps aux | grep VoiceInput | grep -v grep

# 應該看到 4-5 個進程在運行
\`\`\`

## 🎯 如果還是看不到

### 可能原因 1：視窗在其他桌面（Space）
**解決方法**：
1. 使用 `Control + 左/右箭頭` 切換桌面
2. 或使用 Mission Control 查看所有桌面

### 可能原因 2：視窗太小或在螢幕外
**解決方法**：
\`\`\`bash
# 刪除視窗位置設定
rm -rf ~/Library/Application\ Support/voice-input-app/window-state.json

# 重新啟動應用程式
\`\`\`

### 可能原因 3：舊的設定檔
**解決方法**：
\`\`\`bash
# 完全重置
pkill -9 VoiceInput
rm -rf ~/Library/Application\ Support/voice-input-app
open /Applications/VoiceInput.app
\`\`\`
